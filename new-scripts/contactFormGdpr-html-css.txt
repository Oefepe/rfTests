================================================================================
contactFormGdpr.js - HTML + CSS Requirements
================================================================================

DESCRIPTION:
Two-part GDPR-compliant contact form. Part 1 validates fields and GDPR consent.
Part 2 creates the contact via API after validation passes.

================================================================================

================================================================================
DATA ATTRIBUTES:
================================================================================

#contactFormContainer:
  - data-campaign: Campaign ID (required, must not be 'YOUR_CAMPAIGN_ID')
  - data-label: Label/tag ID (optional, can be empty)
  - data-redirect: Redirect URL after successful submission (optional)

================================================================================
URL PARAMETERS:
================================================================================

Required:
  - userId
  - resourceId

Example: ?userId=123&resourceId=456

================================================================================
TWO-PART FLOW:
================================================================================

Part 1 (Validation):
  - Listens for button click
  - Validates all fields including GDPR consent checkbox
  - Sets window.validationComplete = true
  - Calls Part 2

Part 2 (Contact Creation):
  - Only runs if window.validationComplete is true
  - Reads campaign/label from data attributes
  - Submits to API
  - On success: redirects or clears form (including GDPR checkboxes)

================================================================================
NOTES:
================================================================================

- GDPR consent checkbox (id="gdprConsent") is REQUIRED
- Marketing consent checkbox is optional
- Form fields are cleared after successful submission (if no redirect)
- Button shows spinner during submission
- Email format is validated with regex
